# API Request

## Sanctions Screening API

<br />


<Container>
  <Row>
    <Column>

      sanctions and other potential risk.

### Request Object
<Container>
  <Row>
    <Column >
      **Attributes**
      <hr />
      
      **```apiKey```** string REQUIRED\
      Your organization's unique API key. 
      ___

      **```minScore```** integer\
      The minimum similarty score of results to be returned. In other words, only
      matches with a [similarty score](/glossary#similarity-score) equal to or greater than ```minScore``` will be included
      in the result set.\
      **Must be between 80 and 100.**\
      A ```minScore``` of 95 is recommended. 
      ___

      **```sources```** list of strings\
      Which sanction lists to screen against. An array of sanction list ID codes.\
      For example, a value of ```["SDN", "EU"]{:json}``` will screen against the OFAC SDN and EU Financial Sanctions lists.\
      See the [**Data Sources**](/datasources) page for available lists and their codes. 
      ___

      **```types```** list of strings\
      Entity type filter. Only these entity types will be screened. \
      Accepted values: ```person```, ```organization```, ```vessel```, ```aircraft```. 
      ___

      **```cases```** list\
      List of Case items.
      <details>
        <summary>**Case Object**</summary>
        <hr />
        
        **```name```** string REQUIRED\
        Full name of the entity being screened.
        ___

        **```externalId```** string *RECOMMENDED*\
        And ID or index number that maps this case to a search result when 
        screening a batch of cases. 

        ___
        
        **```type```** string\
        The type of this entity.\
        Accepted values: ```person```, ```organization```, ```vessel```, ```aircraft```. 
        ___

        **```dob```** string\
        Date of birth.\
        Format: ```YYYY-MM-DD```
        ___

        **```citizenship```** string\
        Citizenship
        ___

        **```nationality```** string\
        Nationality
        ___

        **```gender```** string\
        Gender
        ___

        **```phoneNumber```** String\
        Phone number
        ___

        **```emailAddress```** string\
        Email address
        ___

        **```cryptoId```** string\
        Crypto wallet ID
        ___

        **```address```** Address
        <details>
          <summary>**Address Object**</summary>
          <hr />
          
          **```address1```** String\
          Address line 1
          ___

          **```address2```** String\
          Address line 2
          ___

          **```phoneNumber```** String\
          Phone number
          ___

          **```city```** String\
          Address city
          ___

          **```stateOrProvince```** String\
          State or province
          ___

          **```postalCode```** String\
          Postal code
          ___

          **```country```** String\
          Country
        </details>

        **```identification```** List of ```Identification```
        <details>
          <summary>**Identification Object**</summary>
          <hr />

          **```type```** string\
          ID type
          ___

          **```idNumber```** string\
          ID number

        </details>

      </details>
      
    </Column>
